# 🌍 I18n (Internationalization)

[⬅️ Back](./README.md) | [🏠 Docs Root](../README.md)

The `i18n` module provides integration with `aiogram-i18n` and a custom manager for storing user locales in FSM (Redis).

---

## 🧠 FSMContextI18nManager

The `FSMContextI18nManager` is a locale manager that stores the user's selected language in FSM storage instead of a database. This approach is faster and requires no additional database queries.

### Features:
- **FSM Storage:** Saves the `locale` key in the user's state.
- **Auto-detection:** Falls back to the Telegram `language_code` if no locale is set in FSM.
- **Allowed Locales:** Restricts supported languages to a predefined list.

---

## 💻 Usage Example

To use the `FSMContextI18nManager`, you need to install the `i18n` extra:
`pip install codex-bot[i18n]`

```python
from aiogram_i18n import I18nMiddleware
from aiogram_i18n.cores import FluentRuntimeCore
from codex_bot.engine.middlewares.i18n import FSMContextI18nManager

# 1. Initialize the manager
manager = FSMContextI18nManager(
    allowed_locales=["ru", "en"],
    default_locale="en"
)

# 2. Setup the middleware
i18n = I18nMiddleware(
    core=FluentRuntimeCore(path="locales/{locale}"),
    manager=manager,
    default_locale="en",
)

# 3. Register in Dispatcher
i18n.setup(dp)
```

---

## 🛠️ Locale Determination Priority

1. **FSM Storage:** If the `locale` key exists in the user's state.
2. **Telegram Language:** If the user's `language_code` is in the `allowed_locales` list.
3. **Default Locale:** Fallback to the `default_locale` specified in the manager.
