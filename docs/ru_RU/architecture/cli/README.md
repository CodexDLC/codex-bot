# CLI — Движок генерации и управления

CLI (Command Line Interface) в **codex-bot** — это мощный инструмент автоматизации, который сопровождает проект на всех этапах: от инициализации до добавления новых функций. Он гарантирует, что ваш проект всегда следует стандартам архитектуры фреймворка.

---

## 💎 Философия CLI

Главная задача CLI — избавить разработчика от рутинного создания файлов и папок ("boilerplate"). Мы следуем трем принципам:
1. **Профессиональный старт**: Проект сразу готов к работе с БД, Redis и i18n.
2. **Изоляция**: Каждая фича создается в своей папке со всеми необходимыми слоями (Logic, UI, Handlers).
3. **Безопасная интеграция**: CLI уважает ваш существующий код и не перезаписывает важные файлы проекта.

---

## 🏗 Основные команды

| Команда | Назначение | Особенности |
| :--- | :--- | :--- |
| **`startproject`** | Создание нового бота | Интерактивный визард, настройка стека, режим Smart Merge. |
| **`create-feature`** | Добавление бизнес-фичи | **Интерактивный режим**: поможет выбрать тип (Telegram/Redis) и название. |
| **`inspect`** | Инструмент отладки | Проверка токена и получение информации о боте напрямую через API. |

---

## 🚀 Интерактивность и Гибкость

Команды генерации (`startproject` и `create-feature`) поддерживают два режима работы:

1. **Интерактивный (Wizard)**: Просто запустите команду без аргументов. CLI задаст необходимые вопросы, предложит варианты выбора и проверит введенные данные.
2. **Командный (Arguments)**: Для опытных пользователей или скриптов автоматизации все параметры можно передать через флаги (например, `--name my_feature --type redis`). В этом случае вопросы задаваться не будут.

---

## 🎨 Режим Smart Merge

Если вы запускаете `codex-bot startproject` в директории, где уже есть проект (например, Django или FastAPI), CLI переходит в режим **Smart Merge**:
- Файлы конфигурации (`pyproject.toml`, `manage.py`, `.env`) создаются с суффиксом `.bot`.
- Ваш бот изолируется в папке `src/{bot_name}`.
