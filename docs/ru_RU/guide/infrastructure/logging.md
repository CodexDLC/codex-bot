# 📝 Логирование (Loguru)

[⬅️ Назад к списку гайдов](../best_practices.md)

Мы рекомендуем использовать библиотеку `loguru` вместо стандартного `logging`. Она проще в настройке, поддерживает ротацию файлов "из коробки" и делает логи читаемыми.

## 🛠 Настройка и использование

```python
from loguru import logger

# 1. Настройка в main.py
logger.add(
    "logs/bot.log",
    rotation="10 MB",
    retention="10 days",
    level="INFO",
    format="{time} {level} {message}"
)

# 2. Использование в оркестраторе
logger.info(f"User {user_id} entered feature {feature_name}")

# 3. Логирование ошибок с трейсбеком
try:
    await some_api_call()
except Exception as e:
    logger.exception("API call failed") # Автоматически добавит трейсбек
```

## 💎 Преимущества подхода
1. **Простота**: Не нужно настраивать хендлеры и форматтеры вручную.
2. **Читаемость**: Цветной вывод в консоль и понятные сообщения.
3. **Ротация**: Автоматическое управление размером файлов логов.
4. **Трейсбеки**: Метод `logger.exception` — это спасение при отладке асинхронного кода.
